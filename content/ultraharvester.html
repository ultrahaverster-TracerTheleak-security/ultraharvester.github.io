Description

UltraHarvester is a comprehensive OSINT framework built for professional security researchers and penetration testers. It combines multiple reconnaissance techniques — email harvesting, DNS enumeration, port scanning, metadata extraction, breach detection and web discovery — into a single unified tool with both CLI and web dashboard interfaces.

Use this tool when you need a full-scope passive and active recon report on a target domain, or when assessing the external attack surface of your own organization.

Legal disclaimer: UltraHarvester is intended for authorized security testing only. Only run it against systems and domains you own or have explicit written permission to test.

Module Description
📧 Email Harvester Collect emails from Google, Bing, DuckDuckGo, Yahoo; SMTP verification; LinkedIn/Twitter/GitHub profile discovery.
🌐 DNS Enumeration Subdomain brute-force + passive recon (crt.sh, HackerTarget); full DNS records; zone transfer checks; WHOIS.
🔌 Port Scanner Multi-threaded TCP scanning; banner grabbing; technology detection; CMS fingerprinting (WordPress, Joomla, Drupal…).
🗂️ Metadata Extractor Google Dorks automation; public document discovery (PDF, DOCX, XLSX); metadata extraction (author, dates, software).
🔑 Leak Checker HaveIBeenPwned API; GitHub credential leak search; Pastebin/paste site monitoring; k-anonymity password check.
🖼️ Web Discovery Website crawler; SSL cert enumeration (crt.sh); Wayback Machine history; Shodan/Censys integration; IP/ASN lookup.
🤖 AI Engine Risk scoring (0–100); finding correlation; GPT-4 executive summary; Telegram/Slack notifications.
📊 Output JSON, CSV, HTML, PDF export; interactive web dashboard; relation graph (domain → subdomain → email → person).

Installation

OPTION 1 — pip install (recommended)
# Clone the repository
git clone https://github.com/yourname/ultraharvester
cd ultraharvester

# Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate      # Linux / macOS
venv\Scripts\activate.bat      # Windows

# Install dependencies and package
pip install -r requirements.txt
pip install -e .
OPTION 2 — Run directly
pip install -r requirements.txt
python main.py --help
CONFIGURE API KEYS
# Copy the example env file and add your keys
cp .env.example .env
nano .env
04

Cheatsheet

SYNTAX
ultraharvester scan [domain] -m [modules] -o [output_dir] -f [formats]
EXAMPLE — Full scan
ultraharvester scan example.com
EXAMPLE — Select modules + HTML output
ultraharvester scan example.com -m emails,dns,ports -f json,html,pdf
EXAMPLE — Custom threads, port range, proxy
ultraharvester scan example.com --ports 1-65535 -t 100 --proxy http://127.0.0.1:8080
EXAMPLE — DNS only / Email only
ultraharvester dns example.com
ultraharvester emails example.com
EXAMPLE — Launch web dashboard
ultraharvester web
# Then open http://localhost:5000 in your browser
05

Options / Flags

Flag Description
-m Comma-separated list of modules to run (emails, dns, ports, metadata, leaks, web, ai)
-o Output directory for results (default: ./output/)
-f Export formats: json, csv, html, pdf
-t Number of threads for port scanning (default: 50)
--ports Port range to scan, e.g. 1-65535 or 80,443,8080
--proxy Proxy URL (supports HTTP and SOCKS5)
--config Path to a YAML configuration file
--shodan-key Shodan API key (overrides .env)
--hibp-key HaveIBeenPwned API key (overrides .env)
--openai-key OpenAI API key for AI summary (overrides .env)
-v Verbose output
